Github Pages

The first post!

I had heard about Github Pages during an event at Noisebridge. Github Pages allows you to make a site that is hosted directly from your repository. This made a lot of sense for me since I was looking for an easy option to showcase what I was working on. I have a JSFiddle account for prototyping and showing off features, but for polished work, it should be hosted somewhere.

Why Github Pages is awesome

  1. It's free!
  2. It's easy to setup
  3. It is connected to a github repo.
  4. You choose which branch is live and when you push to it, your site is updated instantly (or a few minutes tops).
  5. You can connect it to a custom domain.

A bit more on the setup

I won't go into too much detail about the setup, they make it incredibly easy. Start here, I highly recommend that you watch the video. I followed the directions for an organization site using the command line. For step 3, I added my own html, css and js files so that my first commit resulted in this site (mostly) as it appears now.

Next I needed to setup for the custom domain. Check out the quick start guide. I had purchased my domain years ago so I jumped to step 2. For step 3 I used the apex and subdomain instructions. I would recommend this options, unless you have a custom subdomain. This is the only section I could forsee someone having issues with, since you'll need to do these things through your DNS provider. I'm using AWS, so I was pretty comfortable changing the A record and CNAME, but I could see this step being a bit of a nail biter if you haven't done it before. After completing this step it took about 30 minutes for my site to go live, but these things can take up to a day.

And you're done! Enjoy your new site!

For my next post, I'll be recapping the setup of Manatee.Foundation, where we'll be using a similar setup to my site but with the addition of Jekyll, which allows for easy blogging using markdown.
